# Template Analysis For Movie Recommendation System

---

## Comparison of GitHub Project Templates

| Template | Columns/Workflow | Automation Features | Agile Suitability |
|----------|----------------|--------------------|------------------|
| Basic Kanban | To Do, In Progress, Done | No automation | Suitable for simple projects but lacks efficiency |
| Automated Kanban | To Do, In Progress, Done | Automatically moves issues when status changes | Highly suitable for Agile development and sprint tracking |
| Bug Triage | New, Investigating, Fixed | Auto-labeling bugs | Good for maintenance but not full development |
| Team Planning | Backlog, Ready, In Progress, Done | Supports planning but less automation | Useful for teams but less flexible |

---

## Selected Template: Automated Kanban

The **Automated Kanban template** was selected because it supports Agile workflows through built-in automation. Issues automatically move between columns based on their status, reducing manual effort and improving workflow efficiency.

This template aligns well with the Movie Recommendation System because:
- It supports continuous development and iteration
- It integrates well with GitHub Issues created in Assignment 6
- It improves task tracking and visibility

Automated Kanban ensures that development tasks move smoothly across stages, which is essential for managing sprint-based development.
