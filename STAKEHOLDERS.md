# Stakeholder Analysis — Movie Recommendation System

## Stakeholder Analysis Table

| Stakeholder | Role | Key Concerns | Pain Points | Success Metrics |
|------------|------|--------------|-------------|-----------------|
| User (Viewer) | Watches movies and receives recommendations | Wants accurate, relevant, and fast recommendations | Difficult to find good movies manually; too many choices | Receives personalized recommendations within ≤2 seconds with at least 85% relevance |
| Admin | Manages system users, data, and system settings | System reliability, user management, and data accuracy | Manual monitoring is time-consuming; difficult to track system issues | Reduce system errors by 30% and manage users within ≤5 seconds per operation |
| Data Scientist | Develops and improves recommendation algorithms | Model accuracy and quality of recommendations | Poor data quality affects recommendations; model tuning is complex | Achieve at least 85% recommendation accuracy and improve model performance by 5% after updates |
| System Administrator (IT) | Maintains system infrastructure and deployment | System uptime, performance, and security | System crashes or slow performance under heavy load | Maintain 99% system uptime and response time under ≤2 seconds |
| Content Manager | Manages movie data (genres, descriptions, metadata) | Accurate and up-to-date movie information | Manual data updates are slow and error-prone | Update movie records within ≤3 seconds with 95% data accuracy |
| Business Owner | Oversees the system and ensures business value | User engagement and system profitability | Low user retention if recommendations are poor | Increase user retention by 20% and engagement by 25% |
| Marketing Team | Uses system data to understand user preferences and trends | Access to user behavior insights for campaigns | Limited access to user analytics and trends | Generate user reports within ≤5 seconds and improve campaign targeting accuracy by 20% |
| External Data Provider (Kaggle / MovieLens) | Provides datasets for training the recommendation model | Proper usage and integration of data | Data inconsistency and integration challenges | Successfully integrate datasets with ≥95% data completeness and consistency |
