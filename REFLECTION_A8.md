# Reflection — Assignment 8

## Challenges Faced

One of the main challenges in this assignment was deciding the level of detail to include in the state and activity diagrams. Including too many states made diagrams complex, while too few reduced clarity.

Another challenge was aligning diagrams with functional requirements and user stories. It required careful mapping to ensure consistency across assignments.

Designing activity diagrams with decision points and parallel actions also required deeper thinking about system behavior.

## Comparison

State diagrams focus on how individual objects change over time, while activity diagrams focus on workflows and processes.

State diagrams are useful for modeling object behavior, while activity diagrams are better for understanding system operations.

## Lessons Learned

This assignment improved my understanding of the system behavior modeling. It showed me how are the dynamic aspects of a system can be visualized clearly.

## Conclusion

Overall, this assignment strengthened my ability to model system interactions and workflows, preparing for implementation.
