# Reflection 

## Challenges Faced

One of the main challenges in this assignment was translating functional requirements into use cases. It required careful analysis to ensure that each use case accurately represented system behavior and aligned with stakeholder needs.

Another challenge was writing detailed use case specifications. Defining preconditions, postconditions, and alternative flows required structured thinking and attention to detail.

Developing test cases was also challenging because each test case had to be clearly linked to a requirement and include measurable expected results.

## Lessons Learned

This assignment improved my understanding of how the requirements are transformed into system interactions using use cases. It also showed the importance of structuring use cases clearly to support system design.

Additionally, I learned how to create effective test cases that validate both functional and non-functional requirements.

## Conclusion

Overall, this assignment strengthened my understanding of system modeling and validation. It demonstrated how the use cases and test cases work together to ensure that system requirements are correctly implemented.
