# Kanban Board Explanation For Movie Recommendation System

---

## What is a Kanban Board?

A Kanban board is a visual project management tool used to track tasks as they move through different stages of development.

It uses columns such as "To Do", "In Progress", and "Done" to represent workflow stages.

---

## How the Kanban Board Works

In this project, the Kanban board is used to manage tasks from Assignment 6. Each task is represented as a GitHub Issue and moves across columns based on progress.

---

## Workflow Visualization

The Kanban board visualizes workflow by organizing tasks into columns:
- To Do: Tasks that have not started
- In Progress: Tasks currently being worked on
- Testing: Tasks being tested before completion
- Blocked: Tasks that cannot proceed due to issues
- Done: Completed tasks

This makes it easy to track progress and identify bottlenecks.

---

## Work-in-Progress (WIP) Limits

The board limits work-in-progress by ensuring that only a few tasks are in the "In Progress" column at a time (e.g., 3 tasks). This prevents overload and ensures focus on completing tasks before starting new ones.

---

## Agile Principles Support

The Kanban board supports Agile principles by:
- Enabling continuous delivery of features
- Allowing flexibility and adaptation
- Providing real-time visibility of work
- Encouraging collaboration and accountability

---

## Customization

The board was customized by adding:
- Testing column → ensures quality assurance before completion
- Blocked column → helps identify and resolve issues quickly

These additions improve workflow management and align with real-world software development practices.
